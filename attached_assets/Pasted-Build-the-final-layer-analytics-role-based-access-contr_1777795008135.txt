  ROLE-BASED ACCESS CONTROL
  ═══════════════════════════════════════════════════════
  Implement on BOTH frontend and backend:
  
  Role → Allowed modules (sidebar shows only these):
  
  SUPER_ADMIN: ALL modules, ALL properties
  HR_MANAGER: Employees, Recruitment, L&D, Settings (users only)
  OPERATIONS_MANAGER: Properties, Residents, Complaints, Laundry, Communications
    (only their assigned property unless SUPER_ADMIN)
  PROCUREMENT_MANAGER: Vendors, Indents, Purchase Orders, GRN, Inventory
  KITCHEN_MANAGER: Recipes, Menu Planning, Kitchen dashboard, Inventory (read-only)
  PROJECTS_MANAGER: Property Leads, Finance (read-only), Procurement (read-only)
  PROPERTY_ACQUISITION: Property Leads only
  FINANCE: Residents (ledger + payments), Finance dashboards, Procurement (read-only)
  SALES_EXECUTIVE: Sales CRM (own leads only), Property Leads (read-only)
  WARDEN: Properties (own only), Residents (own property), Complaints (own property),
    Laundry (own property), Communications (view + send to own property)
  VENDOR_RESTRICTED: [future phase — placeholder in sidebar]
  AUDIT_READONLY: All modules, read-only (no create/edit/delete buttons shown)
  
  Frontend implementation:
  - usePermissions() hook: returns { canView, canCreate, canEdit, canDelete } for current route
  - Wrap every action button with permission check
  - Sidebar filters navigation items based on role
  - Route guard: redirect to /403 if user navigates to unauthorized route
  - 403 page: friendly 'Access Denied' page with back button
  
  Backend implementation:
  - authorize() middleware on all routes: accepts allowed roles array
  - Property scoping: if user has propertyId, all queries filtered to their property
    (except SUPER_ADMIN and roles with multi-property access)

  NOTIFICATION SYSTEM
  ═══════════════════════════════════════════════════════
  Create Notification model in Prisma:
    id, userId, title, body, type, link, isRead, createdAt
  
  Types of auto-generated notifications:
  - COMPLAINT_SLA_BREACH: to assignedTo user + their manager
  - PAYMENT_OVERDUE: to operations manager of that property (generated daily at 9am)
  - LEAVE_APPROVAL_PENDING: to manager when employee applies leave
  - INDENT_APPROVAL_PENDING: to procurement manager when indent submitted
  - LOW_STOCK: to procurement manager when item below min stock
  - DOCUMENT_EXPIRY: to admin when vendor compliance doc expiring in 30 days
  - LEASE_RENEWAL: to property acquisition when property agreement expiry < 60 days
  
  Implementation:
  - createNotification(userId, {title, body, type, link}) utility function
  - Call this from relevant service functions (complaint SLA checker, etc.)
  - GET /api/notifications → returns last 20 for current user
  - PATCH /api/notifications/:id/read → mark read
  - PATCH /api/notifications/read-all → mark all read
  - Bell icon in topbar: live unread count (polling every 30s with React Query refetchInterval)
  - Notification dropdown: list of notifications, click → navigate to link, mark as read
